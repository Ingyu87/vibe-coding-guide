---

## 실행 원칙

- 한 파일은 한 번에 한 에이전트만 작업한다
- 완료 표시가 없는 파일은 미완료로 간주한다
- 이전 배치가 전부 완료되어야 다음 배치를 시작한다
- 핵심개념요약.md는 1단계 완료 직후 생성하며, 이후 모든 에이전트에 공통 제공한다

## 에이전트 컨텍스트 스펙

### 공통 (모든 에이전트)

```
1. CLAUDE.md 전문
2. 핵심개념요약.md (1단계 완료 후 생성됨 — 1단계 에이전트는 제외)
3. toc_detail.md 중 해당 장 섹션만 발췌
4. 출력 파일 경로 명시
```

### 예외

| 에이전트 | 추가 컨텍스트 |
|---|---|
| 1단계 (프롤로그·0·1장) | 핵심개념요약.md 없음 (아직 미생성) |
| 3단계 파트3 도입부 | 2단계 결과물 파일 전체 목록 전달 |
| 5단계 에필로그 | 00_프롤로그.md 전문 전달 |

## 에이전트 프롬프트 템플릿

각 에이전트 실행 시 아래 템플릿을 사용한다.

```
너는 [장 제목] 집필 전담 에이전트다.

[컨텍스트]
- CLAUDE.md의 모든 집필 규칙을 따른다
- 핵심개념요약.md의 개념과 용어를 일관되게 사용한다
- toc_detail.md의 [해당 장 섹션]에 명시된 구조를 반드시 지킨다

[제약]
- 담당 파일 외 다른 원고 파일은 읽지 않는다
- toc_detail.md에 없는 내용을 임의로 추가하지 않는다
- 분량은 A4 기준 8~12쪽을 목표로 한다

[출력]
- 파일 경로: 03_원고/[파일명].md
- 완료 시 파일 상단 첫 줄에 아래 표시를 추가한다:
  <!-- 상태: 완료 | 작성일: YYYY-MM-DD -->
```

## 완료 상태 관리

### 파일 상단 상태 표시 (3단계)

```markdown
<!-- 상태: 미시작 -->
<!-- 상태: 진행중 | 작업도구: Claude Code -->
<!-- 상태: 완료 | 작성일: 2026-06-03 | 작업도구: Claude Code -->
```

### 배치 완료 조건

해당 배치의 모든 파일에 `상태: 완료` 표시가 있어야 다음 배치를 시작한다.

## Codex 인계 프로토콜

Claude Code 일일 한도 도달 시 아래 순서로 인계한다.

### 인계 전 확인

1. 이 파일(workflow.md)의 상태 컬럼을 현재 상태로 업데이트한다
2. `상태: 진행중` 파일은 삭제 후 Codex가 처음부터 재작성한다
3. `상태: 완료` 파일은 Codex가 건드리지 않는다

### Codex에게 전달할 것

```
- 00_제작기준/CLAUDE.md
- 00_제작기준/toc_detail.md (해당 장 섹션)
- 00_제작기준/핵심개념요약.md (생성된 경우)
- 00_제작기준/workflow.md (현재 진행 상태 확인용)
- 담당 파일 경로
```

### Codex 프롬프트 템플릿

```
Claude Code가 작업 중단되어 인계받는다.
workflow.md의 상태 컬럼을 보고 미시작·진행중 파일만 작업한다.
완료 표시된 파일은 수정하지 않는다.
나머지 집필 규칙은 Claude Code와 동일하게 CLAUDE.md를 따른다.
```

## 핵심개념요약.md 작성 기준

1단계 완료 직후 아래 항목을 담아 `00_제작기준/핵심개념요약.md`를 생성한다.

```
- 이 자료집의 핵심 주장 (프롤로그에서 확정된 것)
- Human in the Loop / Teacher in the Loop / Student in the Loop 정의
- '교사답게'의 의미 (0장에서 확정된 것)
- 반복 사용할 핵심 문장 3~5개
- 이 자료집에서 피해야 할 표현 목록
```
