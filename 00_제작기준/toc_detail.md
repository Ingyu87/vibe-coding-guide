## 프롤로그. AI를 잘 쓰는 교사보다, 교사답게 쓰는 교사

**목적** 독자가 이 자료집을 읽어야 하는 이유를 여는 부분이다.  
**핵심 주장** AI를 잘 쓰는 것보다 중요한 것은 교사답게 쓰는 것이다.

**작성 방향**
- AI 기술 소개로 시작하지 말고 교실 장면으로 시작한다
- 같은 AI 도구라도 어떤 교사는 학생 사고를 살리고, 어떤 교사는 AI가 학생 사고를 대신하게 만드는 대비를 보여 준다

**넣을 내용**
- 잘 쓰는 교사와 교사답게 쓰는 교사의 차이
- 수업 맥락, 학생 성장, 관계, 안전, 책임
- 이 책은 금지 목록이 아니라 판단 기준이라는 안내

**피해야 할 것**
- "AI 시대가 왔다"는 범용 도입부
- 기술 예찬이나 기술 공포만 강조하는 서술
- 코딩 실력 중심의 서론

**예시 장면**
한 교사는 AI로 멋진 퀴즈앱을 만들지만 성취기준과 학생 수준을 놓친다. 다른 교사는 덜 화려하지만 학생이 왜 틀렸는지 설명하게 하는 도구를 만든다.

**체크리스트**
- [ ] 이 자료집의 문제의식이 선명한가?
- [ ] 교사다운 AI 활용의 의미가 드러나는가?
- [ ] 독자가 자신의 수업 장면을 떠올릴 수 있는가?

**마무리 문장 예시**
> AI가 할 수 있다는 사실은, 교사가 해도 된다는 뜻이 아니다.

## Part 1 — 총론

### 0장. 왜 '바이브코딩 하지마세요'인가

**목적** 무분별한 바이브코딩을 경계하고, 만들기 전에 멈추어 판단하는 기준을 세운다.  
**핵심 주장** '하지마세요'는 금지가 아니라 교육적 멈춤의 요청이다.

**작성 방향**
- 선언문처럼 힘 있게 쓰되, 막연한 경고가 아니라 학교 현장에서 생길 수 있는 실제 문제를 중심으로 쓴다
- 마지막에는 그래도 교사가 해야 하는 이유와 도움받을 자료를 연결한다
- 분량 목표: 10~15쪽

**넣을 내용**
- 무분별한 바이브코딩의 폐해 (교육적 철학 없는 일회성 웹앱)
- 바이브코딩을 꼭 해야 하는 것 vs. 하지 않는 것이 더 나은 것 (표/도식)
- 그럼에도 교사가 해야 하는 이유
- Resource: 자료집, 사례집, 연수 자료, 학운위 심의 가이드

**피해야 할 것**
- 바이브코딩 자체를 전면 부정하는 표현
- 위험만 나열하고 대안을 주지 않는 방식
- 기술 유행 비판으로만 끝나는 서술

**예시 장면**
교사가 해커톤에서 만든 앱이 수업 목표와 맞지 않고, 만든 사람만 쓸 수 있으며, 학기 말에는 링크와 데이터가 방치되는 상황.

**체크리스트**
- [ ] 이 앱은 꼭 만들어야 하는가?
- [ ] 기존 에듀테크나 오프라인 활동이 더 나은가?
- [ ] 학생 데이터와 책임이 어디에 남는가?
- [ ] 사용 종료 계획이 있는가?

**마무리 문장 예시**
> 좋은 교사개발자는 더 많이 만드는 사람이 아니라, 만들지 않아도 되는 것을 알아보는 사람이다.

### 1장. 루프와 책임(Human in the Loop)

**목적** 교사, 학생, AI의 역할과 책임 위치를 정리한다.  
**핵심 주장** AI 안에 교사가 있다는 것은 교사의 책임이 흐려지는 것이 아니라 더 분명해지는 것이다.

**작성 방향**
- Human in the Loop를 교육 언어로 번역한다
- 기술 용어 설명보다 교사가 어디서 목표를 정하고, 학생이 어디서 사고를 남기고, 교사가 어디서 확인해야 하는지에 초점

**넣을 내용**
- Teacher in the Loop
- Student in the Loop
- Teacher & Student in the Loop
- 교사개발자가 알아야 할 기본 용어 (→ 부록 카드로 연결)

**피해야 할 것**
- 루프 개념을 이론적으로만 설명하는 방식
- 용어 설명이 본문을 압도하는 구성
- 교사의 책임을 AI 검토 한 번으로 축소하는 서술

**예시 장면**
학생이 AI 챗봇에게 질문하고 답을 받는 활동에서, 교사는 챗봇의 역할과 금지 범위를 설계하고 학생은 AI 도움을 받은 부분을 기록한다.

**체크리스트**
- [ ] 교사 개입 지점이 명확한가?
- [ ] 학생 사고 과정이 남는가?
- [ ] AI의 역할과 한계가 설명되는가?
- [ ] 최종 판단 책임이 교사에게 남는가?

**마무리 문장 예시**
> AI 안에 교사가 있다는 말은, 교사가 사라지는 것이 아니라 더 정확히 남는다는 뜻이다.

**마무리 문장 예시**
> 안전한 앱은 문제가 없는 앱이 아니라, 문제가 생겼을 때 멈추고 설명하고 고칠 수 있는 앱이다.

## Part 3-1 — 수업용 도구

### 수업 설계 도구 만들기

**이 도구는 무엇을 돕는가**
성취기준을 수업 목표로 바꾸고, 학생 수준과 예상 오개념을 확인하며, 활동 순서와 발문, 평가계획을 정리하는 데 도움을 준다.

**교육적 철학은 어디에 들어가는가**
AI가 수업을 대신 설계하는 것이 아니라 교사의 수업 의도를 더 분명하게 만드는 것이다.

**이 도구에서 막아야 할 위험**
맥락 없는 수업안, 학생 수준을 벗어난 발문, 오개념을 강화하는 설명, 평가와 연결되지 않는 활동.

**반드시 포함해야 할 기술 조건**
성취기준 입력 / 학생 수준 입력 / 예상 오개념 입력 / 활동 시간 제한 / 교사 수정 기록 / 출처 확인 / 저장 범위 설정

**개발 고려사항**
- AI가 바로 수업안을 만들기 전에 빠진 조건을 질문하게 한다
- 교사가 수정한 내용이 남도록 한다
- 학생 개인정보 없이 학급 특성을 일반화해 입력하도록 안내한다

**교사라면 꼭 넣을 한 줄**
> AI가 수업을 대신 설계하지 말고, 교사가 성취기준과 학생 맥락을 더 정확히 말하도록 질문하게 해 주세요.

**체크리스트**
- [ ] 성취기준과 수업 목표가 입력되는가?
- [ ] 학생 수준과 예상 오개념을 반영하는가?
- [ ] 교사가 초안을 수정하고 확정하는 단계가 있는가?
- [ ] 출처와 개념 정확성을 확인할 수 있는가?

**마무리**
수업 설계 도구는 빠른 지도안을 만드는 도구가 아니라, 교사가 수업의 이유와 흐름을 더 선명하게 정리하도록 돕는 도구여야 한다.

### 학생용 이미지 생성 도구 만들기

**이 도구는 무엇을 돕는가**
학생이 배운 개념을 시각적으로 표현하고, 자신의 의도와 수정 이유를 설명하도록 돕는다.

**교육적 철학은 어디에 들어가는가**
AI가 학생의 표현을 대신 완성하지 않도록 하는 것이다. 학생이 먼저 아이디어를 내고, AI 도움을 받은 부분과 최종 선택 이유를 남기도록 설계해야 한다.

**이 도구에서 막아야 할 위험**
성적 표현, 폭력, 혐오, 친구 조롱, 얼굴 합성, 개인정보 노출, 실존 인물 악용, 저작권 침해, 역사 왜곡, 과학 오개념.

**반드시 포함해야 할 기술 조건**
학생용 금지 기준 안내 / 프롬프트 기록 / 생성 이미지 기록 / 교사 승인 / 공개 범위 제한 / 신고·삭제 기능 / 닉네임 또는 임시코드 사용

**교사라면 꼭 넣을 한 줄**
> AI 이미지는 학생의 설명을 대신하지 말고, 학생이 왜 그렇게 표현했는지 말하게 하는 자료로 쓰이게 해 주세요.

**체크리스트**
- [ ] 학생이 먼저 기획한 내용이 남아 있는가?
- [ ] 초상권, 폭력, 성적 이미지, 조롱 이미지 기준이 있는가?
- [ ] 교사가 결과물을 확인하고 삭제할 수 있는가?
- [ ] 공개 범위와 신고 기능이 있는가?

**마무리**
학생용 이미지 생성 도구는 표현을 넓힐 수 있지만, 학생의 사고를 대신하거나 친구에게 상처를 주는 순간 교육적 의미를 잃는다.

### AI 챗봇 만들기

**이 도구는 무엇을 돕는가**
학생이 막혔을 때 힌트를 얻고, 자신의 생각을 말로 풀고, 반론을 들어보고, 개념을 다른 표현으로 확인하도록 돕는다.

**교육적 철학은 어디에 들어가는가**
정답을 주는 봇이 아니라 생각을 돕는 대화를 만드는 것이다.

**이 도구에서 막아야 할 위험**
정답 전체 제공, 과제 대필, 개인정보 발화, 상담·심리 문제에 대한 부적절 대응, 폭력·성적·혐오 대화, 할루시네이션.

**반드시 포함해야 할 기술 조건**
챗봇 역할 지시문 / 답변 범위 제한 / 금지 주제 처리 / 개인정보 입력 경고 / 대화 로그 정책 / 교사 모니터링 / 신고 기능 / RAG 자료 범위 확인

**교사라면 꼭 넣을 한 줄**
> 학생에게 정답을 대신 주지 말고, 학생이 자기 생각을 한 문장 더 말하도록 질문하는 챗봇으로 설계해 주세요.

**체크리스트**
- [ ] 챗봇의 역할과 답변 범위가 명확한가?
- [ ] 정답 전체 제공과 과제 대필을 제한하는가?
- [ ] 교사가 대화 흐름을 확인할 수 있는가?
- [ ] 학생에게 모니터링 목적을 안내하는가?

**마무리**
AI 챗봇은 학생의 질문 창구가 될 수 있다. 그래서 답변의 편리함보다 대화의 안전과 사고의 보존을 먼저 설계해야 한다.

### 조작도구(인터랙티브 학습) 만들기

**이 도구는 무엇을 돕는가**
수학 도형, 과학 실험, 사회 지도, 언어 카드, 분류 활동, 순서 배열, 퀴즈와 미션형 학습게임.

**교육적 철학은 어디에 들어가는가**
조작이나 게임 자체가 아니라 조작 뒤의 언어화와 학습 내용의 적용이다.

**이 도구에서 막아야 할 위험**
오개념 강화, 접근성 문제, 조작만 남는 활동, 과한 게임성, 보상 중독, 순위 경쟁.

**반드시 포함해야 할 기술 조건**
조작 기록 / 되돌리기 / 교사용 해설 / 제한 조건 표시 / 반응형 화면 / 키보드 대체 조작 / 오답 후 재도전 / 과도한 랭킹 제한

**교사라면 꼭 넣을 한 줄**
> 학생이 화면을 만지는 데서 끝나지 않고, 무엇을 바꾸었고 왜 결과가 달라졌는지 말하게 해 주세요.

**체크리스트**
- [ ] 조작 과정이 기록되는가?
- [ ] 학생이 예측과 설명을 남기는가?
- [ ] 오개념을 막는 교사용 해설이 있는가?
- [ ] 과한 게임성과 순위 경쟁을 제한하는가?

**마무리**
조작도구와 학습게임은 흥미를 만들 수 있지만, 재미가 학습을 대신하면 교육적 의미를 잃는다.

### 평가·피드백 도구 만들기

**이 도구는 무엇을 돕는가**
루브릭 기반 피드백, 자기평가, 오개념 점검, 성장 기록 정리, 서술형 피드백 초안.

**교육적 철학은 어디에 들어가는가**
점수를 자동으로 내는 것이 아니라 피드백의 질을 높이는 것이다.

**이 도구에서 막아야 할 위험**
AI가 점수나 등급을 확정하는 것, 편향된 기준으로 학생을 판단하는 것, 낙인 표현, 이의 제기 불가 구조.

**반드시 포함해야 할 기술 조건**
루브릭 표시 / 피드백 근거 표시 / 교사 승인 / 학생 이의 제기 또는 수정 요청 / 자동 점수 확정 금지 / 기록 보관 기간 설정

**교사라면 꼭 넣을 한 줄**
> AI가 학생을 평가하게 하지 말고, 교사가 더 나은 피드백을 줄 수 있도록 근거와 표현을 정리하게 해 주세요.

**체크리스트**
- [ ] 교사 최종 판단 단계가 있는가?
- [ ] AI 활용 유무에 따른 안내가 있는가?
- [ ] 루브릭과 피드백 근거가 표시되는가?
- [ ] 학생의 이의 제기나 질문 절차가 있는가?

**마무리**
평가·피드백 도구의 목표는 더 빠른 채점이 아니라 더 나은 성장 안내다.

## Part 3-2 — 행정업무용 도구

### 학급 운영 프로그램 만들기

**이 도구는 무엇을 돕는가**
학급 회의, 역할 분담, 상담, 생활지도, 프로젝트 운영, 학급 일지, 공지 관리.

**교육적 철학은 어디에 들어가는가**
학생을 관리 대상으로 보는 것이 아니라 학급 공동체의 참여자로 대하는 것이다.

**이 도구에서 막아야 할 위험**
학생 점수화, 위험군 분류, 상담·가정·정서 정보 과다 기록, 학급 기록의 학생 간 노출.

**교사라면 꼭 넣을 한 줄**
> 학급 운영 도구는 학생을 관리하기보다 학급 구성원이 더 안전하게 참여하고 협의하도록 돕게 해 주세요.

**체크리스트**
- [ ] 운영 목적이 학생에게 설명되는가?
- [ ] 민감한 생활지도 기록을 최소화하는가?
- [ ] 자동 낙인이나 위험도 점수화가 없는가?
- [ ] 삭제, 수정 요청, 열람 권한 기준이 있는가?

**마무리**
학급 운영 프로그램은 학급을 편하게 관리하는 도구가 아니라 관계와 참여를 더 잘 돌보는 도구여야 한다.

---

### 업무 도움 프로그램 만들기

**이 도구는 무엇을 돕는가**
생기부, 세특, 공문, 회의록, 가정통신문, 보고서, 연수 공문 정리.

**교육적 철학은 어디에 들어가는가**
업무를 AI에게 넘기는 것이 아니라 교사의 관찰과 판단을 더 명료한 문서로 표현하게 하는 것이다.

**이 도구에서 막아야 할 위험**
AI가 공식 사실을 만들어내는 것, 공문이나 회의록에 없는 내용을 추가하는 것, 학생 기록을 과장하는 것, 상담·성적·가정 정보가 외부로 전송되는 것.

**교사라면 꼭 넣을 한 줄**
> AI가 공식 문서를 대신 쓰게 하지 말고, 교사가 확인한 근거를 바탕으로 초안을 정리하게 해 주세요.

**체크리스트**
- [ ] 원문 근거와 AI 초안이 구분되는가?
- [ ] 자동 발송이나 자동 제출이 제한되어 있는가?
- [ ] 민감정보 입력 경고가 있는가?
- [ ] 교사 최종 확인 단계가 있는가?

**마무리**
업무 도움 프로그램은 교사를 덜 책임지게 하는 도구가 아니다. 교사가 더 정확하고 덜 소진되게 일하도록 돕는 도구여야 한다.

## 에필로그. 덜 만들고, 필요한 만큼만 쓰는 교사

> 교사다운 바이브코딩은 더 많이 만드는 일이 아니라, 학생에게 필요한 만큼만 만들고 책임 있게 거두는 일이다.
