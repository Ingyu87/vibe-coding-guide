## 2. 각 장 집필 공통 구조

Part 3의 모든 도구 장은 아래 순서를 반드시 지킨다.

```
이 도구는 무엇을 돕는가
교육적 철학은 어디에 들어가는가
이 도구에서 막아야 할 위험          ← 핵심 위험 박스
반드시 포함해야 할 기술 조건
개발 고려사항
교사라면 꼭 넣을 한 줄              ← 반복 박스
체크리스트                           ← □ 판단 질문형
마무리
```

Part 1·2·4의 장은 아래 순서를 따른다.

```
목적
핵심 주장
작성 방향
넣을 내용
피해야 할 것
예시 장면
체크리스트
마무리 문장 예시
```

## 3. 문체·어말어미 기준

| 용도 | 어말어미 | 예시 |
|---|---|---|
| 본문 설명 | ~다 / ~한다 / ~필요하다 | 수집하지 않는 것이 가장 안전한 설계다. |
| 교사 실천 안내 | ~해 주세요 / ~확인해 주세요 | 교사가 성취기준과 학생 맥락을 더 정확히 말하도록 질문하게 해 주세요. |
| 체크리스트 | ~있는가? / ~되는가? / ~확인했는가? | 학생 수준과 예상 오개념을 반영하는가? |
| 핵심 문장 | 짧고 단정한 선언형 | 공유는 링크를 보내는 일이 아니라 책임 범위를 정하는 일이다. |

**피할 표현**
- ~하면 좋을 것 같다 / ~할 수도 있다
- 아마, 대체로, 적당히
- 과한 감탄형, 기술 홍보 문구

## 4. 문장·문단 길이

- 한 문장: 40~60자 내외
- 긴 설명은 두 문장으로 나눈다
- 한 문단: 2~4문장
- 체크리스트: 항목당 질문 하나

---

## 5. 용어 표기 기준

**처음 등장 시 병기**
- Human in the Loop(사람이 과정 안에 들어가는 구조)
- Teacher in the Loop / Student in the Loop

**권장 표기**
바이브코딩 / 생성형 AI / AI 챗봇 / 웹앱 / 프롬프트 / PRD / implementation.md / RAG / API / 개인정보처리방침 / 학교운영위원회(학운위)

**주의할 표기**
- 인터렉티브 → **인터랙티브**
- 부적절 대회 → **부적절 대화**
- 학습운영 → **학급 운영**

## 6. 반복 박스 (각 장에 반드시 사용)

모든 장에 아래 박스를 포함한다. 박스는 독자가 장을 빠르게 훑어보고 핵심을 찾게 해 준다.

| 박스명 | 용도 |
|---|---|
| **핵심 위험** | 이 도구에서 막아야 할 것 |
| **교사라면 꼭 넣을 한 줄** | 교육적 설계의 핵심 한 문장 |
| **체크리스트** | □ 판단 질문형 항목 |
| **만들지 않는 것이 나은 경우** | 제작 중단 판단 기준 |

**체크리스트 작성법**
```
권장 (판단 질문형)
□ 성취기준과 수업 목표가 입력되는가?
□ 교사가 최종 수정하고 확정하는 단계가 있는가?

피할 것 (행동 지시형)
□ 성취기준 입력
□ 교사 확인
```

## 7. 강조·편집 기준

- 핵심 문장만 **굵게**
- 위험 기준은 표나 박스로
- 밑줄·빨간색·느낌표·굵은 글씨 남용 금지
- 폰트: 맑은 고딕 (대체: Noto Sans CJK KR)
- 줄간격: 본문 1.15~1.3 / 문단 뒤 4~6pt / 체크리스트 항목 2~4pt

## 8. 전체 톤 기준

- 경고하지만 겁주지 않는다
- 권하지만 부추기지 않는다
- 기술을 설명하지만 기술 중심으로 흐르지 않는다
- 교사를 통제하지 않고 판단을 돕는다
- 학생을 보호하되 학생의 주체성을 빼앗지 않는다
