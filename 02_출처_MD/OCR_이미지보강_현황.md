# OCR·이미지 확인 보강 현황

## 결론

현재 환경에는 Tesseract OCR 엔진이 없어 자동 OCR은 수행하지 못했다. 대신 PDF 텍스트 추출에서 빈 페이지로 잡힌 페이지의 내장 이미지를 추출했고, PDF별 contact sheet를 만들어 이미지 확인 기반 보강이 가능하도록 했다.

## 생성된 보강 자료

- 빈 페이지 이미지 인덱스: `_empty_page_images/README.md`
- contact sheet 목록: `_empty_page_images/CONTACT_SHEETS.md`
- contact sheet 이미지 폴더: `_empty_page_images/_contact_sheets`

## 확인된 상태

| 항목 | 상태 |
|---|---|
| PDF 원문 Markdown 변환 | 10개 PDF 모두 완료 |
| 텍스트 추출 오류 | 발견되지 않음 |
| 빈 페이지 이미지 추출 | 완료 |
| contact sheet 생성 | 완료 |
| 자동 OCR | 미수행, OCR 엔진 없음 |
| 이미지 수동 확인 | 대표 페이지 확인 완료, 전체 페이지는 추가 확인 필요 |

## 이미지 확인 예시

### `바이브코딩을 부탁해(26. 5. 23.).pdf`

텍스트 추출에서 비어 있던 페이지 중 실제로는 핵심 슬라이드가 포함되어 있었다.

- p.10: 민간 개발자와 교사 개발자의 핵심 바이브 비교
- p.11~19: 교사 개발자 윤리 7대 원칙
- p.30: Gemini Canvas 또는 Canva AI 사용 흐름 화면

이 자료는 원고 작성 시 다음 장에 반영할 수 있다.

- 프롤로그: 교사 개발자의 정체성과 민간 개발자와의 차이
- Part 1 0장: 교사 개발자는 기능 구현보다 학습 몰입, 업무 경감, 교육용 콘텐츠를 우선한다는 관점
- Part 2 보안·윤리: 학생 성장, 역할 경계, 개인정보, 신중한 검토, 공공성, 출처 존중, 투명성
- Part 3 AI 챗봇: 상담 챗봇 등에서 저장·공유 여부 안내와 안전 장치 필요

### `디버깅_강의_v3.6 (배포용).pdf`

텍스트 추출에서 비어 있던 일부 페이지는 실제로 오류 설명과 코드 예시 이미지였다.

- p.10: JavaScript 대소문자 구분으로 발생하는 `TypeError` 예시
- 오류 원인, 수정 위치, 코드 일부를 함께 보여 주는 자료로 확인됨

이 자료는 원고 작성 시 다음 장에 반영할 수 있다.

- Part 3 개발 고려사항: 오류 메시지와 코드 위치를 함께 확인하는 습관
- Part 4 적용 이후: 배포 후 오류 대응, 새 대화방에서 맥락 정리, 수정 기준
- 부록: AI에게 오류를 전달할 때 전체 코드, 오류 메시지, 원하는 결과를 함께 제공하는 템플릿

### `바이브코딩_심화원고.pdf`

이 자료는 대부분 이미지 중심 PDF로 보인다. 텍스트 추출은 거의 비어 있지만, contact sheet에서 슬라이드 흐름은 확인된다.

확인되는 흐름은 다음과 같다.

- 본격적인 개발 도구 사용 고민
- 프론트엔드, 백엔드, 데이터베이스 개념
- HTML, CSS, JavaScript, Python, Node.js 기초 개념
- IDE와 CLI 비교
- Google Antigravity 설치와 사용
- AI 코딩 작업, 프로젝트 실행, 배포 흐름
- Firebase, 데이터베이스, 사용자 인증, 저장 구조
- AI 활용 한계와 주의점

이 자료는 원고 작성 시 다음 장에 반영할 수 있다.

- Part 2 설치형·웹형·디벗: 개발 환경과 실행 환경 확인
- Part 3 도구별 개발 고려사항: 프론트엔드, 백엔드, 데이터베이스, 인증, 저장 구조
- Part 4 성장 로드맵: 입문 이후 심화 개발 흐름

## 다음 보강 방식

자동 OCR이 필요하면 Tesseract 또는 다른 OCR 엔진 설치가 필요하다. 현재 상태에서는 다음 방식이 가장 안전하다.

1. contact sheet로 빈 페이지 전체 흐름을 먼저 본다.
2. 중요한 슬라이드만 개별 이미지를 열어 사람이 확인한다.
3. 확인 내용을 원본 Markdown의 해당 페이지 아래에 `이미지 확인 보강` 섹션으로 추가한다.
4. 원고 작성 시 이 보강 내용은 원문 인용이 아니라 판단 기준, 예시 장면, 체크리스트로 재구성한다.
